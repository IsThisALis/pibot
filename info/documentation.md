# Архитектура и функциональность Pibot

## Структура проекта

```
pibot/
├── bot-data/                   # Данные, загружаемые при старте
│   ├── phrases.json            # Триггер-фразы и ответы
│   ├── rp-phrases.json         # RP-команды (ответом на сообщение)
│   ├── banned-users.json       # ID заблокированных пользователей
│   ├── personality.md          # Системный промпт для AI
│   └── personality-public.md   # Публичная версия промпта
├── env/                        # Неотслеживаемые конфиги (gitignored)
│   ├── telegram-token          # Токен бота
│   ├── groq-key                # API ключ Groq
│   └── dev-ids.json            # ID разработчиков
├── info/                       # Документация
│   ├── command-list.md         # Список команд (вывод по "пибот команды")
│   ├── documentation.md        # Эта документация
│   ├── botinfo.md              # Вывод по "пибот био"
│   └── changelog.md            # Вывод по "пибот обновы"
├── source/
│   └── pibot.py                # Основной код бота (~970 строк)
├── pyproject.toml              # Конфигурация mypy
├── setup.sh                    # Скрипт развёртывания
└── launchbot.sh                # Скрипт запуска
```

## Обработка сообщений (`handle_message`)

1. **Hard block** — если ID пользователя есть в `banned-users.json`, сообщение игнорируется полностью
2. **Age filter** — сообщения старше `MAX_MESSAGE_AGE` (120 секунд) игнорируются, кроме начинающихся с "пибот"
3. **Команды** — если текст начинается с `пибот `, ищется подкоманда в `PIBOT_COMMANDS`, проверяется ранг или dev-статус
4. **RP-команды** — если сообщение является ответом и текст совпадает с RP-командой (без префикса)
5. **Триггер-фразы** — поиск точного совпадения в `phrases.json`
6. **Chance trigger** — фразы, начинающиеся с "пибот инфа", возвращают случайное число
7. **AI-ответ** — если сообщение является ответом и содержит @упоминание бота, вызывается Groq AI

## Система команд

Команды вызываются через префикс `пибот <команда>`.
Регистрируются декоратором `@pibot_command(name, value, dev_only=False)`.
Хранятся в словаре `PIBOT_COMMANDS`.

### Ранговая система

4 уровня доступа:

| Ранг | Название | Кто | Неприкосновенность |
|------|----------|-----|--------------------|
| 1 | Owner | Создатель чата | mute, kick, ban, rank change |
| 2 | Admin+ | Назначается владельцем | mute, kick, ban |
| 3 | Admin | Админы чата по умолчанию | mute, kick, ban |
| 4 | Member | Все остальные | — |

Проверка ранга: `user_rank <= command_value` даёт доступ.

### Команды и их значения

| Команда | Value | Мин. ранг | Описание |
|---------|-------|-----------|----------|
| `кинь в гулаг @user` | 1 | Owner | Telegram-бан + запись в banned-users.json |
| `верни @user` | 1 | Owner | Telegram-анбан + удаление из banned-users.json |
| `ранг n для @user` | 1 | Owner | Изменить ранг (n = 2, 3, 4) |
| `сотри n` | 2 | Admin+ | Удалить n+1 последних сообщений |
| `кикни @user` | 2 | Admin+ | Кик пользователя |
| `мут @user [n]` | 3 | Admin | Мут на n минут (минимум 0.5) |
| `размут @user` | 3 | Admin | Размут |
| `био` | 4 | Member | Информация о боте |
| `обновы` | 4 | Member | Список изменений |
| `команды` | 4 | Member | Список команд |
| `ранги` | 4 | Member | Список пользователей с особыми рангами |
| `заблокируй <id>` | dev_only | DEV_ID | Блокировка пользователя (без Telegram-бана) |

### DEV_ID команды

Команда `заблокируй` доступна только пользователям, чьи ID перечислены в `env/dev-ids.json`.
Ранговая проверка для неё не применяется — только проверка по списку разработчиков.

### Hard block (`banned-users.json`)

`bot-data/banned-users.json` хранит множество ID пользователей.
При обработке любого сообщения бот проверяет, не заблокирован ли отправитель:
- `кинь в гулаг` — добавляет ID в список + Telegram-бан
- `верни` — удаляет ID из списка + Telegram-анбан
- `заблокируй` — добавляет ID в список без Telegram-бана
- Заблокированные пользователи полностью игнорируются (hard block)

## AI-функции

- Используется только Groq (Llama 3.3 70B) через OpenAI-совместимый API
- Системный промпт загружается из `personality.md`
- При упоминании @bot_username в ответе на сообщение бот отправляет контекст в LLM
- Rate limit: 3 вызова в минуту на чат

## Антиспам

### RateLimiter (глобальный)
5 сообщений в секунду, токен-бакет. При превышении сообщение дропается.

### Age filter
Сообщения старше 120 секунд игнорируются (не актуальны при перезапуске).
Команды с "пибот" не блокируются этим фильтром.

### Trigger phrase spam filter
Если пользователь отправляет более 5 одинаковых триггер-фраз за 60 секунд:
- Бот отвечает "Ой всё"
- Триггер-фразы этого пользователя игнорируются 2 минуты

### Telegram mute
При спаме (быстрые сообщения, не триггер-фразы) — `restrictChatMember` на 60 секунд.
Админы, Admin+ и Owner не подвержены спам-фильтру.

## RP-система

При ответе на сообщение текстом из `rp-phrases.json` бот заменяет `{mention1}` на автора, `{mention2}` на цель. Пример:
- A: "обнять" (отвечая B) → "A нежно обнял B"

## Чанковая отправка

Сообщения длиннее 4000 символов разбиваются на части и отправляются последовательно с задержкой.

## Типизация (mypy)

Проект аннотирован типами. Конфигурация в `pyproject.toml`.
Некоторые ошибки от PTB (union-attr, arg-type, index) отключены через `--disable-error-code`,
так как код корректно обрабатывает None на уровне runtime.
